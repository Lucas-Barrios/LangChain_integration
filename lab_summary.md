# Lab Summary: MCP vs Direct API Trade-offs

MCP (Model Context Protocol) and direct API integration both connect AI agents to external tools, but they make different trade-offs. MCP standardizes the interface so one client library works with any compliant server — tools are discovered dynamically at runtime, and swapping a data source requires only a config change, not new code. This portability is its core value for multi-source agents or teams reusing agents across projects. The cost is added indirection: each tool call crosses a subprocess or HTTP boundary (10–50ms overhead), and debugging spans two layers (agent + server). Direct API integration — wrapping calls manually as LangChain `BaseTool` subclasses — gives full control over auth, retries, and error shaping with no extra process overhead, but every new data source requires a custom implementation, which doesn't scale as the number of tools grows. Use MCP when the tool ecosystem is broad or likely to change; use direct integration for narrow, stable, performance-sensitive use cases where you own both sides of the interface.
